# T.P.2-Objetos-Concesionaria
Este proyecto simula el funcionamiento de una **concesionaria de vehículos** (autos y motos) utilizando **Java** con un enfoque totalmente orientado a objetos. La aplicación carga una lista fija de vehículos y, mediante el uso del **Collections Framework** y la **Stream API**, genera distintos reportes por consola: vehículo más caro, más barato, ordenamientos por precio y ordenamiento natural (por marca, modelo y precio).
- Clases abstractas e interfaces.
- Herencia y polimorfismo.
- La interfaz `Comparable` y comparadores personalizados (`Comparator`).
- Programación funcional con la Stream API.
- Buenas prácticas de código reduciendo el boilerplate con **Lombok**.

## 🎯 Consigna del trabajo práctico

> Desarrollar un programa con Java que simule el funcionamiento de una concesionaria de vehículos. Al ejecutarse, la aplicación debe generar exactamente la salida por consola especificada por la cátedra, cargando los vehículos en un único método (sin solicitar datos por pantalla), implementando al menos una clase abstracta, una interfaz, y utilizando el framework Collections y la API Stream.
